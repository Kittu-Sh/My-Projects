Overview of My Work
This week, I focused on building a fully responsive landing page from scratch. The main goal was to ensure the site looks professional on both my laptop and my phone while keeping everything "offline-ready" by managing my own project files and images locally.

My Technical Journey
1. Designing the Blueprint (Figma)
I started by creating a wireframe in Figma. I wanted to map out how the "Hero" section would transition to mobile. I chose a light blue tint for the main image because it makes the white text much easier to read—a small detail that really improved the look.

2. Writing the Code (HTML & CSS)
I used Semantic HTML5 to make the code organized and easy to read. The CSS was the most time-consuming part. I used Flexbox for the layout and added a Media Query breakpoint at 768px to fix the mobile view.

I fixed an issue where the navbar was wider than the phone screen by using overflow-x: hidden.

I also added a :hover and :active effect to the links so they turn blue when you click them, providing better feedback for the user.

3. Adding Interactivity (JavaScript)
I didn't use any libraries; I wrote the Vanilla JS myself to handle the mobile menu toggle. It took some trial and error to get the hamburger icon (☰) to switch to an "X" when clicked, but it works perfectly now.
